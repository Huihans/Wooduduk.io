Problem Solving

증상이 아니라 원인에서 끝냅니다

🐛 고스트가 호스트로 뽑혀 멀티가 시작되지 않던 버그 · 우드득

상황
특정 사람만 멀티가 시작되지 않고 무한 대기. 재현이 불규칙했습니다.
추적
로그를 붙여 추적 → 호스트를 ‘접속자 중 가장 작은 uid’로 뽑는데, 정원을 채우려 넣은 AI 대역(고스트)이 후보에 섞여 있었습니다.
원인
실제 유저 uid가 사전순으로 고스트보다 뒤면 고스트가 호스트로 뽑히고, 고스트는 시작 시각을 기록하지 않아 나머지가 영원히 대기.
해결
증상 우회 대신 선출에서 고스트를 제외하고 ‘실유저 없는 방은 파기’ 안전망 추가 → uid 순서에 좌우되던 현상이 완전히 사라짐.
배움
재현이 어려운 버그일수록 증상이 아니라 원인을 찾아 끝내는 것이 재발을 막습니다. (기술문서에 트러블슈팅 4건 정리)

🐛 클라우드 세이브를 복원할 때 데이터를 못 찾을 수 있던 문제 · 운랜디

상황
PlayFab 클라우드 세이브로 유저 데이터를 복원하는데, 증강(Augment) 데이터를 id로 매칭해 찾는 구조였습니다.
원인
클라이언트 데이터(ScriptableObject)와 서버(PlayFab Title Data)의 id가 어긋나면 복원 시 해당 데이터를 못 찾았습니다. .asset 수십 개의 id를 손으로 채우다 보면 어긋나기 쉬웠습니다.
해결
클라·서버 id를 하나의 기준으로 맞추고, 사람이 실수할 여지를 에디터 스크립트로 id를 자동 부여해 원천 차단했습니다. (본인 담당: PlayFab 서버 데이터 파트)
